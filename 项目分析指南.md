## 项目概览

### 🎯 这个项目是做什么的？

**一句话总结**：这是一个**智能旅行助手**，用户可以像跟人聊天一样问它问题，比如：

- "北京明天天气怎么样？"
- "查一下9月13日从北京到上海的火车票"
- "周杰伦最近有演唱会吗？"

系统会自动理解你的问题，去数据库里查数据，然后用自然语言回答你！

### 🏗️ 整体架构（用餐厅比喻）

想象你在一家**智能餐厅**吃饭：

| 角色         | 项目中的对应                  | 功能            |
| ---------- | ----------------------- | ------------- |
| 👤 顾客      | 用户                      | 提问："我要一份北京天气" |
| 🧑‍💼 前台接待 | Streamlit 界面 + 路由 Agent | 听懂问题，决定派谁去做   |
| 🍳 天气厨师    | 天气 Agent (A2A)          | 专门负责查天气       |
| 🎫 票务厨师    | 票务 Agent (A2A)          | 专门负责查票        |
| 📦 食材仓库    | MCP 服务器                 | 管理数据库，提供数据    |
| 🏪 食材库     | MySQL 数据库               | 存放天气、票务数据     |

## 核心概念通俗解释

### 1️⃣ MCP (Model Context Protocol)

#### 📖 一句话定义

MCP 就像一个**通用的"数据服务员"协议**，规定了怎么从数据库里取数据给 AI 用。

#### 🍔 生活化例子

想象你去麦当劳：

- **以前**：每个店员都要自己去后厨拿汉堡，效率低
- **现在**：有个专门的"取餐窗口"（MCP），所有店员都通过这个窗口取餐，标准化、效率高

#### 💻 在项目中的体现

看 `mcp_server/` 目录下的文件：

- `mcp_weather_server.py`：天气数据服务员
- `mcp_ticket_server.py`：票务数据服务员

**核心作用**：

- 把复杂的数据库查询封装成简单的"工具"
- AI 不需要懂 SQL，只要说"查一下北京天气"就行
- 所有数据查询都通过 MCP 进行，统一管理

***

### 2️⃣ A2A (Agent-to-Agent)

#### 📖 一句话定义

A2A 就是**多个 AI 助手之间分工合作**的框架，就像公司里不同部门的员工协作。

#### 🏢 生活化例子

想象一家旅行社：

- **前台接待**：听懂客户需求
- **天气专员**：只负责查天气
- **票务专员**：只负责查票
- **财务专员**：只负责算账

客户来了，前台听懂需求后，派给对应的专员去做，最后汇总结果给客户。

#### 💻 在项目中的体现

看 `a2a_server/` 目录下的文件：

- `weather_server.py`：天气专家 Agent
- `ticket_server.py`：票务专家 Agent

**核心优势**：

- **分工明确**：每个 Agent 只做一件事，做得更好
- **易于扩展**：想加新功能？加个新 Agent 就行
- **并行处理**：多个 Agent 可以同时工作，更快

***

### 3️⃣ Agent (智能体)

#### 📖 一句话定义

Agent 就是一个**有特定技能的 AI 助手**，能听懂指令、会用工具、能完成任务。

#### 🤖 生活化例子

想象你的手机助手：

- 你说："明天天气怎么样？"
- 它：1️⃣ 理解你的问题 2️⃣ 调用天气 API 3️⃣ 整理结果 4️⃣ 用自然语言回答你

这就是一个 Agent！

#### 💻 在项目中的体现

项目里有这些 Agent：

1. **路由 Agent**：决定用户的问题该给谁处理
2. **天气 Agent**：专门处理天气查询
3. **票务 Agent**：专门处理票务查询

***

### 4️⃣ LangChain

#### 📖 一句话定义

LangChain 就是一个**AI 应用的"乐高积木"套装**，提供了很多现成的组件，让你快速拼出 AI 应用。

#### 🧱 生活化例子

想象你要搭一个机器人：

- **以前**：你得自己做轮子、做手臂、写控制程序，很难
- **现在**：用乐高积木（LangChain），轮子、手臂、控制器都有现成的，拼起来就行

#### 💻 在项目中的体现

项目用 LangChain 来：

- 连接大语言模型（DeepSeek）
- 创建提示词模板
- 管理对话历史
- 调用工具

***

### 5️⃣ Streamlit

#### 📖 一句话定义

Streamlit 就是一个**超级简单的网页制作工具**，用 Python 写几行代码就能做出漂亮的网页界面。

#### 🎨 生活化例子

想象你要开个 lemonade stand（柠檬水摊）：

- **以前**：你得自己搭帐篷、做招牌、摆桌子，很麻烦
- **现在**：有个现成的摊位套装（Streamlit），打开就能用，还很漂亮

#### 💻 在项目中的体现

看 `main.py`：

- 用 Streamlit 做了聊天界面
- 用户可以在网页上输入问题
- 系统的回答会显示在网页上

### 🎬 1. main.py - 主入口文件

**作用**：用户直接交互的网页界面

**关键功能**：

- 创建漂亮的聊天界面（用 Streamlit）
- 接收用户的问题
- 把问题发给路由 Agent
- 显示 Agent 的回答

**类比**：餐厅的前台，负责接待顾客

***

### ⚙️ 2. config.py - 配置文件

**作用**：存放所有配置信息，就像一个"设置面板"

**里面有什么**：

```python
# 大模型配置
api_key = "..."          # DeepSeek API 密钥
api_url = "..."          # API 地址
model_name = "..."       # 使用的模型

# 数据库配置
db_host = "localhost"    # 数据库地址
db_user = "root"         # 数据库用户名
db_password = "123456"   # 数据库密码

# 天气 API 配置
weather_api_key = "..."  # 和风天气 API 密钥
```

**为什么需要配置文件**：

- ✅ 敏感信息（密码、密钥）集中管理
- ✅ 修改配置不用改代码
- ✅ 不同环境（开发/生产）用不同配置

***

### 🤖 3. a2a\_server/ - A2A Agent 目录

这里住着两个"专家"：

#### 🌤️ weather\_server.py - 天气专家

**职责**：

- 只处理天气相关的问题
- 调用天气 MCP 服务器查数据
- 把数据整理成自然语言回答

**工作流程**：

```
用户问："北京明天天气怎么样？"
    ↓
天气 Agent 收到
    ↓
调用 MCP 天气服务："查北京的天气"
    ↓
MCP 返回数据
    ↓
天气 Agent 整理回答："北京明天晴，25°C..."
```

#### 🎫 ticket\_server.py - 票务专家

**职责**：

- 只处理票务相关的问题（火车票、机票、演唱会票）
- 调用票务 MCP 服务器查数据
- 把数据整理成自然语言回答

```python

    """查询指定城市的天气数据"""

    # 返回结果
```

#### 🎫 mcp\_ticket\_server.py - 票务数据管理员

**职责**：

- 直接连接 MySQL 数据库
- 提供"查火车票"、"查机票"、"查演唱会票"的工具

**作用**：把数据库里的"原始数据"变成"人类可读的格式"

## 数据流和工作原理

### 🔄 完整的一次问答流程

让我们用一个具体例子来说明：

**用户问**："北京明天天气怎么样？"

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 用户输入问题                                        │
│  main.py (Streamlit 界面)                                    │
│  用户在网页输入："北京明天天气怎么样？"                       │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 路由 Agent 理解并分发                                │
│  路由 Agent 分析问题：                                        │
│  "这是天气查询，派天气专家去处理！"                            │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 天气 Agent 接收任务                                  │
│  weather_server.py (A2A Agent)                                │
│  "好的，我来查天气！"                                          │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 调用 MCP 天气服务                                    │
│  mcp_weather_server.py (MCP Server)                           │
│  "帮我查一下北京明天的天气数据"                                │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 5: 查询 MySQL 数据库                                    │
│  weather_data 表                                               │
│  SELECT * FROM weather_data WHERE city='北京' AND fx_date=... │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 6: 返回数据                                             │
│  数据库 → MCP → 天气 Agent                                    │
│  数据：{city: '北京', temp_max: 25, text_day: '晴', ...}    │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 7: 天气 Agent 整理回答                                   │
│  "北京明天天气晴，最高温度25°C，最低温度15°C..."              │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 8: 显示给用户                                           │
│  main.py (Streamlit 界面)                                    │
│  在网页上显示回答                                              │
└───────────────────────────────────────────────────────────────┘
```

## 🎯 快速上手指南

### 启动顺序（重要！）

这个项目需要启动 5 个服务，按顺序来：

```
1️⃣ 终端 1 - 启动天气 MCP 服务器
   py -3.9 mcp_server/mcp_weather_server.py

2️⃣ 终端 2 - 启动票务 MCP 服务器
   py -3.9 mcp_server/mcp_ticket_server.py

3️⃣ 终端 3 - 启动天气 A2A Agent
   py -3.9 a2a_server/weather_server.py

4️⃣ 终端 4 - 启动票务 A2A Agent
   py -3.9 a2a_server/ticket_server.py

5️⃣ 终端 5 - 启动 Streamlit 主应用
   streamlit run main.py
```

然后打开浏览器访问显示的 URL（通常是 <http://localhost:8501），就可以用了！>
